Methodology & Analysis

1 Data Exploration & Cleaning

  • Loaded and inspected dataset structure (284,807 × 31)
  • Identified and removed 1,081 duplicate rows
  • Verified no missing values in any column
  • All columns were float64 type except Class (int64)

2 Outlier Detection (IQR Method)

  • Applied Interquartile Range (IQR) method on transaction amounts
  • Outliers were identified but not removed (potentially fraudulent transactions)
  • Visualized distribution using boxplots

3 Descriptive Statistics

  • Calculated measures of central tendency and spread
  • Compared normal vs. fraudulent transaction statistics
  • Analyzed correlations between features

4 PCA Component Analysis

The dataset features V1-V28 are results of Principal Component Analysis applied to original transaction data for anonymization. These represent the most important patterns in descending order:

  • V1: Most significant pattern (e.g., regular spending patterns)
  • V2: Second most important (e.g., international vs. domestic transactions)
  • V3-V28: Progressively less important transaction patterns

Key Findings

Extreme Class Imbalance

Only 0.17% of transactions were fraudulent (473 out of 284,807), presenting significant challenges for predictive modeling.

Transaction Amount Analysis

Fraudulent transactions had higher average amounts (€123.87 vs €88.41) but lower medians (€9.82 vs €22.00), indicating some high-value fraud cases.

PCA Transformation Success

The PCA components effectively anonymized sensitive banking data while preserving essential patterns for fraud detection analysis.

# Descriptive statistics comparison
fraud_count = credit_data['Class'].sum()
normal_count = len(credit_data) - fraud_count

print(f"Normal transactions: {normal_count:,} ({normal_count/len(credit_data)*100:.2f}%)")
print(f"Fraud transactions: {fraud_count} ({fraud_count/len(credit_data)*100:.2f}%)")

# Amount comparison
print(f"Normal transactions - Mean: €{credit_data[credit_data['Class']==0]['Amount'].mean():.2f}")
print(f"Fraud transactions - Mean: €{credit_data[credit_data['Class']==1]['Amount'].mean():.2f}")
